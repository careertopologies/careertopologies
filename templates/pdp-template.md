## Dicas para Usar Este Template

1. **Seja específico**: "Melhorar comunicação" é vago. "Apresentar 3 tech talks internas" é específico.
2. **Priorize**: Não tente melhorar tudo de uma vez. Foque em 2-3 gaps principais.
3. **Mensure**: Defina como você saberá que progrediu (métricas, feedback, entregas).
4. **Revise regularmente**: PDI não é documento estático. Ajuste conforme aprende e contexto muda.
5. **Celebre progressos**: Reconheça pequenas vitórias, não apenas o objetivo final.
6. **Peça feedback**: Use 1:1s para discutir progresso e ajustar curso.
7. **Documente aprendizados**: Mantenha registro do que funcionou (e do que não funcionou).
